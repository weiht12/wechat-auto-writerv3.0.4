写作方向说明补丁
修复自定义写作方向后页面显示和滚动问题
问题分析
01 现象描述
点击"自定义写作方向"后，下方的"写作方向说明"区域没有正常显示，且页面无法滚动。
02 根本原因
CSS 中写作方向说明区域的 display 设置为 none，或者父容器的 overflow 设置不当导致无法滚动。
03 影响范围
- "写作方向说明"输入框无法使用
- 页面无法滚动到下方内容
- 用户无法自定义写作风格
修复方案
步骤 1
修改 CSS 显示属性
#写作方向说明 { display: block !important; visibility: visible !important; }
步骤 2
修复容器滚动
.page-write-layout { overflow-y: auto !important; max-height: 100vh; }
步骤 3
检查事件监听
// 确保点击"自定义写作方向"后触发 customDirectionBtn.addEventListener('click', () => { customDirectionArea.style.display = 'block'; window.scrollTo(0, customDirectionArea.offsetTop); });
